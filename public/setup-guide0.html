Setup Guide

From Firebase project → Hosting → Live app
1) Create Firebase project
  1. Go to Firebase Console → Add project.
  2. Enable Authentication (Email/Password) and Realtime Database (in test mode, then add rules below).
2) Add a Web app & SDK config
  1. Project settings → Your apps → Web → Register app.
  2. Copy your firebaseConfig and paste it in public/js/app.js at the top.
3) RTDB rules

Rules example:

{
  "rules": {
    "tenants": {
      "$uid": {
        "kv": {
          "$key": {
            ".read": "auth != null && auth.uid === $uid",
            ".write": "auth != null && auth.uid === $uid"
          }
        }
      }
    },
    "users": {
      "$uid": {
        ".read": "auth != null && auth.uid === $uid",
        ".write": "auth != null && auth.uid === $uid"
      }
    }
  }
}
4) Install Firebase CLI & init Hosting
  1. npm i -g firebase-tools
  2. firebase login
  3. In your project folder (with public/): firebase init hosting
  4. Set public as the public directory; choose single-page app “No”.
5) Deploy
  1. firebase deploy --only hosting
  2. Open your Hosting URL (e.g., https://<project-id>.web.app)
6) Auth redirect domain

Add your domain (e.g., you-6bddf.web.app) to Firebase Auth → Settings → Authorized domains.

7) Turn on Cloud Sync (optional)
  1. In the app, open Settings → Cloud Sync → set to ON.
  2. Click Sync Now to push current local data to RTDB.
